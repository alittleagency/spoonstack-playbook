auth + middleware

cookie check — no Supabase on edge

middleware.ts — final architecture

"/" — always rewrites to public/landing.html. No auth check. Logged-in users see the landing page, not /display.

Public paths — /sign-in, /sign-up, /forgot-password, /reset-password, /pending, /upgrade, /api, /landing.html — all pass through with NextResponse.next().

Protected paths — /display, /brain, /people, /onboarding — check for cookie sb-xyxdcwtcfhhczucddhhj-auth-token. Missing cookie → 307 to /sign-in. Cookie present → pass through. App handles session validation after that.

No createServerClient, no updateSession, no edge crypto. Bundle size dropped from 83kB to 27kB.

hard-won patterns

waitForFunction arg order: fn, undefined, {'{'} timeout {'}'} — NOT fn, {'{'} timeout {'}'} (passes options as arg, defaults to 30s timeout)

BottomSheet detection: [style*="z-index: 100"] catches all widget types including those without inputs.

React controlled inputs: .fill() then toBeEnabled() before clicking — never force:true before React state updates.

Submit buttons in BottomSheet always need {'{'} force: true {'}'} (overlapping divs intercept pointer events).

Content audit: "bella"/"jackson" on /display are intentional FAMILY_SCHEDULE placeholders → skipPages: ["/display"].
